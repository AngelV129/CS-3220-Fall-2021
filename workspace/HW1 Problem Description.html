CS3220 Homework 1

Please deploy your application on CS3, and package your project in a zip file and submit it to Canvas. The open-source software tool 7-Zip can be used to create the zip file. Please also create and submit an HTML file hw1.html which contains a hyperlink to your application on the CS3 server.

A local art teacher has a small business that offers painting lessons for kids. There are two types of lessons: group lessons and individual lessons. Kids taking group lessons are divided into groups with up to 5 kids per group. The kids in a group will attend group lessons together, and they cannot attend lessons given to other groups. Parents pay for their kids' lessons by buying "lesson packs". A lesson pack includes 6 or more lessons - either individual or group but not mixed.

In this semester you are going to develop a web-based system that helps the teacher to manage his (or her) business. In this assignment we will focus on students and groups.

1. Home Page (10pt)

When the application is requested without any parameter, it displays two links (or buttons) like the following:

Students | Groups

Clicking on Students should take the user to the student listing page, and clicking on Groups should take the user to the group listing page.

2. List Student (20pt)

The student listing page displays all the students in the system. For example:

Clicking on New Group takes the user to a page where a new group can be added. For example:

Name

Note that after a new group is added, the application should redirect the user to the group listing page.

5. New Student (35pt)

Clicking on New Student takes the user to a page where a new student can be added. For example:

Name
Birth Year
Parent Name
Parent Email
Group

Note that the dropdown list must be dynamically generated based on the groups currently in the system. Specifically, the dropdown list should include all the groups (including the ones added using the New Group function) that have less than 5 students in the group. The group field for a new student can be empty, which means the student is not in any group, or in other words, the student will only take individual lessons.

Also note that birth year is entered when adding a student, while in the student listing, age is displayed instead of birth year. Age is calculated as (current year - birth year).

After a new student is added, the application should redirect the user to the student listing page.

Notes

  • As I said in class, the vast majority of web applications can be considered as data + operations on the data, so first identify the data in this application and create the corresponding classes (e.g. Student and Group), then identify the operations in this application, e.g. list students, add student, and so on.
  • Create one servlet for each operation. If an operation involves form processing, use doGet() to display the form and doPost() to process the form.
  • To focus on the main functionality, we will assume that the users will always provide proper input - in other words, you do not need to implement any input validation.
  • Changes made by the users (e.g. new students and groups) need to be stored in application scope but do not need to be saved to a file. Those changes will be lost after the application is unloaded or reloaded, and that's OK for this assignment.
  • Do not spend too much time beautifying the user interface - you can do that in later assignments when we use JSP.
